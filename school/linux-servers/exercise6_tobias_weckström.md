How to get help! and exercises for directories and files

            Task 1: man ls command provides a more comprehensive manual of the command and
            its uses, in the manual page that opens, the text doesn’t wrap to the next line so you
            have to navigate using arrow keys to see the full content of each line, and you can also
            navigate up and down using arrowkeys, and navigate down “see more” using
            return(enter) key. There is an option to quit by pressing Q or help by pressing H, the
            “help” opens a “summary of less commands” which the same navigation principles as
            earlier apply. Using ls –help doesn’t give an as comprehensive guide on the uses and I
            am not able to navigate upwards to see the rest of the contents from the beginning.

            Task 2: After logging in, I’m in the home directory and I can tell by the ~ symbol in the cli
            and also by using pwd. In the directory /usr/games I can find 2 files using ls -la. The
            working directory is the directory you are in at that moment, shortcuts to change to home
            directory: cd ~ and simply cd. Using tab to navigate to /var/backups/ was very easy from
            the home directory, /v tab, resulting in /var/ and then /var/b tab to get the whole path.
            The command cd /root does not work since we don’t have permissions.

            shown, the parameter to be used is -n or –number. The `touch` command updates the
            access and modification timestamps of existing files to the current time or creates empty
            files if they don’t exist.
