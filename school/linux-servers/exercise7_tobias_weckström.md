Exercise: vim & nano

            Nano was very easy to use, and all the functionality can be seen on the screen. It is
            ideal for very quick config tweaks and basic syntax/small code, but struggles with larger
            scripts and files since it loads it all into memory.




            When opening vim environment using the vim command, at the bottom you can see the
            file you accessed or just created, how many lines it has, and how many bytes/characters
            it has. Vim feels very intuitive, easy navigating using arrow keys or hjkl keys, press I to
            start inserting text somewhere, alos just pressing normal characters twice seemed to
            work to start inserting, esc to exit insert mode, u to delete what you just wrote(cant be in
            insert mode). Ctrl + R key also redoes what was deleted using u. Typing :12 , or any
            numeric value after the colon, will put you in that line to access and edit it quickly, this is
            very useful for debugging. Typing /word will search for specific text, in this case the word
            “start” is what I used and found it in my text immediately, the search is
            capitalisation/syntax sensitive. Pressing d twice will delete the current line you are on in
            the navigation mode, typing any number before d + d will delete that many lines.
            Pressing o in navigation mode starts a new line, and typing :x, will save the changes and
            exit.
