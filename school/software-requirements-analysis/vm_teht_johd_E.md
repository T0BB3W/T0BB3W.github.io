# Assignment.1.1: Introduction

## Study Objectives

1.  Students can explain the role and importance of Requirements Analysis in an IT project

2.  Students can identify different types of requirements.

3.  Students can name the main stages of Requirements Analysis and\
    explain the objectives and tasks of each stage.

4.  Student can describe the target CASE with personalised user stories / scenarios.

## Tasks

1.  Present in your own words the stages of Requirements Analysis and the objectives and main tasks of each stage. 1 paragraph per stage is sufficient. You may use Google and Internet besides the course materials, but indicate your sources in your document.

2.  Connect a Requirement examples to a corrsponding Requirement type.\
    (It may not always be unambiguous..)

| **Requirement examples** |  | **Requirement type** |
|----|----|----|
| As a student I want to see my study transcript digitally so that I can use it anywhere 24/7. |  | Business Requirement |
| All study credits should be stored in one common platform. |  | User Requirement |
| Recording study credits should be easy and effortless. |  | Functional Requirement |
| The software must be launched latest on 2.3.2023. |  | Quality Requirement |
| The maximum response time is 2 seconds. |  | System or API Requirement |
| The software must be compatible with Firefox and Opera browser versions published after 2022. |  | Constraint |
| Using the software requires signing in with a valid user account and a password. |  |  |
