# Ideas

I didn't know right from the start what I wanted to do, so I tested creating an Ultrasonic Sensor with LCD I2C, which I was able to do surprisingly quickly. After the Ultrasonic Sensor with LCD I2C I wanted to do something with moving parts, so I started to make an ESP32-CAM Pan and Tilt Video Streaming Web Server (2 Axis) project, I got it working but I thought it was boring. So, I started to make a new project which was ESP32-CAM Remote Controlled Car Robot Web Server. I took a model from randomnerdtutorials.com but I made my own connections, for example my robot has four-wheel drive. I tried to get the ultrasonic to work at the end of the project, but I only got it to work when I uploaded the codes separately. If I had done this project from the beginning, I would have made the ultrasonic work.

# What did you need to learn

In general, I think that esp32cam is difficult right from the start, there are a lot of things that you must fight with. Just uploading that code is tedious, especially if you are doing it for the first time. I think I would earn 5, because I made more than 1 robot in the course, I took over the esp32cam, I learned to use the FT232RL FTDI Programmer, C++, HTML, CSS, JS coding and electrical stuff.

I learned a surprising amount in this course, especially with the use of esp32cam, a lot of new things. There was also a lot of tinkering and testing with the electrical connections. Soldering, crafting, using imagery, etc.

I think this course was nice. Of course, I cursed the robot when it didn't work, but then when it did, I felt like a winner.
